Was ist responsive Webdesign?

  • ansprechbar“, „empfänglich“, „zugänglich“, “reaktionsfähig”

  • „reagierendes Webdesign“



  • Mischung aus fluidem und adaptiven Layout

  • Fluides Layout = fließendes Layout

    --> passt sich immer Browserbreite/ Bildschirmgröße an
    → Proportionen bleiben immer gleich

  • Adaptives Layout

    → hat Breakpoints, Layout bleibt fest bis Breakpoints,
    dann passt es sich erst an

  • Responsive Layout

    hat Breakpoints
    zwischen Breakpoints ist Layout flexibel
    → passt sich der Bildschirmgröße an



    Elemente

  • Einzelne Elemente/ Texte/Bilder/Videos flexibel → Größenangaben in Prozentwerten



    • Schriften/Schriftgrößen

  • Webfonts sind auf aktuellen Anforderungen vorbereitet


  • Relative Größenangaben für Schriften (Prozent, em oder rem)


  • Schriftgröße im body-Tag für die verschiedenen Breakpoints definieren, in darunter liegenden Elementen nur relative Größen benutzt


    Farben

  • ausreichende Kontraste

Responsive Webdesign in HTML CSS

  • "mobile first"








Vor- und Nachteile

    Vorteile
  • überall erreichbar (egal ob mit Smartphone oder Laptop…etc)

  • wenig kosten für andere Mobile Ansichten

  • zukunftsfähige Websites, da immer alles mobiler wird

  • Hardware unabhängiges Design

  • geringerer Wartungsaufwand



    Nachteile
  • mehr Aufwand

  • Langsame Ladezeit

  • Ältere Browser verstehen gewisse Anweisungen nicht (Explorer 8)

  • Dauer der Entwicklung erhöht sich

  • Viele kleine Zugriffe
    -->jede Seite erzeugt Overhead an Daten die übertragen werden müssen,
    was oft gerade an Handy’s zu Abbrüchen kommt

Was versteht man unter Adaptive Webdesign

  • übersetzt: „anpassungsfähig“

  • Überbegriff für verschiedene Herangehensweisen, wie man Internet Seiten layouten kann

  • für verschiedene Displaygrößen im Web optimiert

  • festes Gestaltungsraster

  • Kombination mit Media Queries

  • ausgesuchte Layouttyp ist also „fixed“
  • verschiedene Ansichten für die gleiche Interseiten

  • Maße für verschiedene Varianten orientieren sich an bestimmten Geräten

  • Durch Media Queries wird die passende Variante individuell auf den Seitenbesucher geladen

  • Ist Handy ein wenig zu klein oder zu groß, keine optimale Ansicht

Vor- und Nachteile

    Vorteile
  • kann gut mit klassischen Mockups und Skizzen gearbeitet werden, da feste Abmessungen

  • Viel gestalterischer Freiraum (starre Raster)

  • Technisch unkompliziert umzusetzen

  • Zeitsparende Umsetzung

  • Abmessungen müssen nur klar definiert werden, aber nicht voll flexibel sein

  • Kostengünstigere Arbeitsprozesse


    Nachteile
  • häufig nur für bestimmte Geräte optimiert oder entwickelt

  • Häufige Fehlerdartstellung auf unterschiedlichen Geräten

  • Aufwändige Zielgruppenanalyse, um gewisse Geräte zu bestimmen

  • Häufig mehr CSS-Code als man machen müsste

  • fehlende Möglichkeit sich auf zukünftige Geräte einzustimmen

  • starre und unflexible Darstellungsvariante

Unterschiede Adaptive und Responsiv Webdesign

  • bei einem Adaptive Webdesign gibt es feste Layoute

  • die dann auch, auf die verschiedene Anwendungsbereiche genutzt werden

  • Interseiten passen, wenn sie die gleiche Abmessungen besitzen.

  • Responsive Webdesign passt sich immer dem jeweiligen Endgerät anpasst

  • Durch „reagierende“ Verhalten ist eine optimale Darstellung auf verschiedenen Displays möglich.

  • Somit sind auch andere oder zukünftige Geräte damit abgedeckt

  • Adaptiven Layouts wird viel mit Pixeln zur Einstellung von Breitenangaben von Elementen und Container gearbeitet

  • somit entsteht also ein festes Layout.

  • Im Responsive Webdesign werden dort Elemente oder Container nicht in Pixel angegeben, sondern mit Prozenten.

  • Dadurch ist es diesen Layouts möglich meist die komplette Bildschirmbreite auszunutzen

  • somit wird kein freier Platz verschwendet mit zum Beispiel unnötigen Rändern
